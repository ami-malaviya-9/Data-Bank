### Project Overview

This project analyzes operational and transactional data for Data Bank, a neo-bank that operates entirely online and links customer bank balances with cloud data storage allocations.

The goal of this analysis is to evaluate network structure, customer distribution, transaction behavior, and operational efficiency to support business planning and growth.

This case study demonstrates advanced SQL skills in analytics, aggregations, and business intelligence.

### Objectives

Analyze the structure of the Data Bank network

Understand customer distribution across regions

Evaluate transaction patterns and volumes

Measure node migration efficiency

Generate insights to support strategic planning

## Business Questions Answered
### Network Analysis

How many unique nodes exist in the Data Bank network?

How are nodes distributed across regions?

How many customers are located in each region?

### Transaction Analysis

Total transaction amount per region

Unique count and total amount by transaction type

Average number and size of deposits across customers

### Operational Efficiency

Average time required to migrate customers to a new node

Monthly behavior of active customers performing multiple transactions

### Customer Activity Insights

Monthly count of customers making:

More than one deposit

At least one purchase or withdrawal
